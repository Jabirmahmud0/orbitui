# Changelog

All notable changes to this project are documented in this file.

The format is based on Keep a Changelog, and published versions are managed through semantic-release.

## Unreleased

- Documentation and contribution guidance continue to evolve on `main` until the next automated release.

## Release Process

Released entries in this file are generated by the automated release pipeline.

That workflow is responsible for:

- determining the next version from Conventional Commits
- generating release notes
- updating this changelog
- publishing the package to npm
- creating the GitHub release

Until the first tagged release is published, this file serves as the changelog baseline for the project.
